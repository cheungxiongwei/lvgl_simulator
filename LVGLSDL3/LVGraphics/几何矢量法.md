# 几何矢量法 (Geometric Vector Method)

### 数学问题

将矩形视为四个顶点 $V_i$ 构成的集合。通过移动部分顶点并保持其余顶点在特定“导轨”上，求由变形后顶点集合 $V'_i$ 反推出的正交参数。

### 数学推理

1. **顶点展开：**


$$V_i = O + R_\theta \cdot \text{Local}(V_i)$$

2. **物理拉杆模拟：**
选取固定点 $P_{fix}$ 和活动点 $P_{act}$ 构成向量 $\vec{L}$。
计算拉伸标量 $d = \Delta\vec{P} \cdot \frac{\vec{L}}{|\vec{L}|}$。
更新移动顶点：$V'_{act} = V_{act} + d \cdot \frac{\vec{L}}{|\vec{L}|}$。

3. **中心重构：**
计算所有新顶点的重心作为新中心：$O' = \frac{1}{n}\sum V'_i$。

4. **逆向对齐 (Reverse Alignment)：**
应用逆旋转矩阵 $R_{-\theta}$ 将 $V'_i$ 映射回 AABB 空间：
$$V'' = R_{-\theta} \cdot (V' - O')$$
通过 $\max(V'') - \min(V'')$ 获取 $W', H'$。

### 注意事项

* **计算消耗：** 涉及开方（求模长）和多次矩阵变换，开销高于代数法。

* **拓扑维持：** 必须确保平行边在移动后依然平行，否则逆向对齐会产生形状扭曲。

* **直观性：** 代码逻辑与人的视觉直觉高度统一，易于调试。

```cpp
/**
 * @brief 使用几何拉杆模拟法求解旋转矩形拉伸后的新参数
 * * @param base_rect      按下鼠标瞬间记录的原始正交矩形 (x, y, w, h)
 * @param base_transform 按下鼠标瞬间记录的原始变换信息 (包含旋转角度 deg)
 * @param dx             当前鼠标位置相对于按下点的世界坐标 X 偏移
 * @param dy             当前鼠标位置相对于按下点的世界坐标 Y 偏移
 * @param mask           当前的拖拽锚点标识 (例如 ANCHOR_TOP | ANCHOR_LEFT)
 * * @return std::tuple<float, float, float, float> {新x, 新y, 新宽, 新高}
 */
std::tuple<float, float, float, float> solve_edge_geometric(
    LVGraphicsRect base_rect, Transform base_transform, float dx, float dy, anchor_mask_t mask) 
{
    // --- Step 1: 初始化几何状态 ---
    // 以记录的原始中心为旋转中心，计算出矩形在旋转状态下在世界坐标系中的 4 个顶点
    base_transform.set_pivot(base_rect.center());
    auto [tl, tr, bl, br] = base_transform.transform(base_rect);

    // 复制顶点副本，后续将在此副本上施加形变
    lv_point_precise_t new_tl = tl, new_tr = tr, new_bl = bl, new_br = br;

    // --- Step 2: 处理水平方向（左右边）的形变 ---
    // 逻辑：如果 mask 包含 LEFT，则以右侧边为基准向左推拉；如果包含 RIGHT，则以左侧边为基准向右推拉
    if (mask & ANCHOR_LEFT) {
        // 构建从右向左的“水平拉杆”向量 (tr -> tl)
        float v_dx = tl.x - tr.x;
        float v_dy = tl.y - tr.y;
        float len = std::hypotf(v_dx, v_dy); // 原始宽度
        
        // 计算拉杆单位方向向量
        float dir_x = v_dx / len, dir_y = v_dy / len;
        // 计算鼠标位移在拉杆方向上的有符号投影长度 d
        float d = dx * dir_x + dy * dir_y;
        
        // 尺寸退化保护：防止宽度被压缩到 1 像素以下
        if (len + d < 1.0f) d = 1.0f - len;
        
        // 只有左侧的两个顶点（tl 和 bl）会沿着此方向移动
        new_tl.x += dir_x * d; new_tl.y += dir_y * d;
        new_bl.x += dir_x * d; new_bl.y += dir_y * d;
    } 
    else if (mask & ANCHOR_RIGHT) {
        // 构建从左向右的“水平拉杆”向量 (tl -> tr)
        float v_dx = tr.x - tl.x;
        float v_dy = tr.y - tl.y;
        float len = std::hypotf(v_dx, v_dy);
        
        float dir_x = v_dx / len, dir_y = v_dy / len;
        float d = dx * dir_x + dy * dir_y;
        
        if (len + d < 1.0f) d = 1.0f - len;
        
        // 只有右侧的两个顶点（tr 和 br）会沿着此方向移动
        new_tr.x += dir_x * d; new_tr.y += dir_y * d;
        new_br.x += dir_x * d; new_br.y += dir_y * d;
    }

    // --- Step 3: 处理垂直方向（上下边）的形变 ---
    // 逻辑：如果 mask 包含 TOP，则以下侧边为基准向上推拉；如果包含 BOTTOM，则以上侧边为基准向下推拉
    // 注意：如果此时是拖拽角点（如 TOP_LEFT），Step 2 已经移动了顶点，Step 3 将在 Step 2 的基础上叠加位移
    if (mask & ANCHOR_TOP) {
        // 构建从下向上的“垂直拉杆”向量 (bl -> tl)
        float v_dx = tl.x - bl.x;
        float v_dy = tl.y - bl.y;
        float len = std::hypotf(v_dx, v_dy); // 原始高度
        
        float dir_x = v_dx / len, dir_y = v_dy / len;
        float d = dx * dir_x + dy * dir_y;
        
        if (len + d < 1.0f) d = 1.0f - len;
        
        // 只有上方的两个顶点（tl 和 tr）会沿着此方向移动
        new_tl.x += dir_x * d; new_tl.y += dir_y * d;
        new_tr.x += dir_x * d; new_tr.y += dir_y * d;
    } 
    else if (mask & ANCHOR_BOTTOM) {
        // 构建从上向下的“垂直拉杆”向量 (tl -> bl)
        float v_dx = bl.x - tl.x;
        float v_dy = bl.y - tl.y;
        float len = std::hypotf(v_dx, v_dy);
        
        float dir_x = v_dx / len, dir_y = v_dy / len;
        float d = dx * dir_x + dy * dir_y;
        
        if (len + d < 1.0f) d = 1.0f - len;
        
        // 只有下方的两个顶点（bl 和 br）会沿着此方向移动
        new_bl.x += dir_x * d; new_bl.y += dir_y * d;
        new_br.x += dir_x * d; new_br.y += dir_y * d;
    }

    // --- Step 4: 计算物理新中心并执行逆向旋转 ---
    // 此时 new_tl, new_tr, new_bl, new_br 构成了变形后的旋转矩形
    // 计算这四个点在世界坐标系中的几何中心点 O'
    float new_cx = (new_tl.x + new_tr.x + new_bl.x + new_br.x) / 4.0f;
    float new_cy = (new_tl.y + new_tr.y + new_bl.y + new_br.y) / 4.0f;

    // 创建逆向变换矩阵：以新中心 O' 为旋转中心，旋转角取反 (-deg)
    // 这一步的作用是将这个“歪”的矩形整体摆正
    Transform inv_transform;
    inv_transform.deg = -base_transform.deg;
    inv_transform.set_pivot(new_cx, new_cy);

    // 将变形后的顶点映射回正交坐标系（AABB 空间）
    lv_point_precise_t aabb_tl = inv_transform.transform(new_tl);
    lv_point_precise_t aabb_br = inv_transform.transform(new_br);

    // --- Step 5: 计算并返回最终的正交参数 ---
    // 在摆正后的空间里，左上角坐标就是 aabb_tl，宽和高就是对应的差值
    float new_w = aabb_br.x - aabb_tl.x;
    float new_h = aabb_br.y - aabb_tl.y;

    return {aabb_tl.x, aabb_tl.y, new_w, new_h};
}
```
